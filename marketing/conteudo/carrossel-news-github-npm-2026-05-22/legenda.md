# Legenda — GitHub incrementa aprovação do npm

Token vazado em CI = pacote malicioso publicado.

O GitHub anunciou um novo fluxo de publicação no npm com 2FA obrigatório e uma área de preparação (`npm stage publish`). Antes do módulo ir pro registry, ele passa por uma camada extra de aprovação — torna bem mais difícil um ataque automatizado empurrar código comprometido pelos tokens de CI.

Arrasta pro lado pros detalhes.

Me conta nos comentários: você ainda usa `npm publish` direto, ou já usa provenance / 2FA no teu CI?
