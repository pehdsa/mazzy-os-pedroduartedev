Architecture 02 / 08
01 · Comece pelo modelo
Como esses dados se relacionam?
Antes de pensar em qual banco usar, pensa na natureza dos dados que sua aplicação manipula.
Cenário relacional
Dados com integridade forte, transações e relatórios consistentes pedem banco relacional.
PostgreSQL MySQL SQL Server
Exemplo Financeiro, pedidos, faturamento, CRM e controle de estoque rodam melhor em modelo relacional porque dependem de consistência entre tabelas e auditoria de transações.

Architecture 04 / 08
03 · Banco × arquitetura
Casamento com o estilo do sistema
Monólito modular
Relacional resolve bem. PostgreSQL costuma ser um default sólido.
Microserviços
Cada serviço escolhe seu banco conforme o domínio, não conforme o time vizinho.
Event-driven
Banco transacional + broker para integração. Event store quando histórico é parte do modelo.
Analytics
Banco operacional separado de data warehouse ou lakehouse.
Tempo real
Redis pra baixa latência, séries temporais pra métricas, streaming pra fluxos contínuos.
Busca textual
Elasticsearch como motor de busca. Evite como fonte principal da verdade.
