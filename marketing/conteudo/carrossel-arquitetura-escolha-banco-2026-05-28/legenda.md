Banco de dados não se escolhe por popularidade.

Tem dev feliz porque foi de Mongo "que tá em alta". Outro guardando dado crítico no Redis. Outro usando Elasticsearch como fonte da verdade.

Três meses depois, a aplicação trava, os dados começam a sumir, e ninguém entende o que tá rolando.

A real é simples. Banco bom é aquele que sustenta:

→ o formato dos dados
→ o jeito que sua aplicação acessa
→ as garantias que a arquitetura precisa

Vertical ou horizontal? SQL ou NoSQL? Documento ou key-value?

Nenhuma dessas perguntas se responde com "qual tá na moda".

Salva esse aqui pra próxima vez que alguém sugerir trocar de banco "porque sim".
